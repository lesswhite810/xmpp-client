## 修复计划

**问题**：`XmppFramingDecoder` 分割 XML 片段后，`<stream:features>` 元素缺少 `stream` 命名空间声明，导致解析失败。

**修改文件**：`StanzaStreamParser.java`

**修改内容**：

1. 在 `parse()` 方法开头添加检测逻辑，判断 XML 是否使用了 `stream:` 前缀但缺少命名空间声明

2. 如果需要命名空间上下文，将 XML 包装在虚拟根元素中：
   ```xml
   <xmpp:wrapper xmlns:stream='http://etherx.jabber.org/streams'>
     [原始XML片段]
   </xmpp:wrapper>
   ```

3. 解析包装后的 XML，然后提取原始元素内容

这种方法可以保留 XMPP 流的命名空间上下文，同时不影响其他正常的 XML 解析。
